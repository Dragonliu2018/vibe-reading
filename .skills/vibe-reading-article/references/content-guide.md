## 代码库

### 阅读顺序
1. `README.md` → 项目定位、核心特性、快速开始
2. `pyproject.toml` / `package.json` / `go.mod` → 版本、依赖、入口
3. 目录树（2 层，不递归）→ 识别分层架构
4. 核心入口文件（`main.py` / `index.ts` / `cmd/main.go` 等）→ 启动流程
5. 按需深入关键模块（不要读所有文件）

### 需提取的信息
- 项目元数据：版本、协议、语言版本、代码量（用 `wc -l **/*.py` 估算）、维护者
- 架构分层：画出分层图（3-5 层），每层职责
- 核心特性：5-8 个，每个对应具体文件/模块
- 数据流：请求从入口到存储的完整路径
- 关键设计模式：观察者、工厂、中间件等

### 推荐文章结构（代码解读）

```
§1 项目简介         — 定位 + 核心特性卡片组（6个左右）
§2 目录结构         — 顶层树 + 核心目录说明
§3 分层架构         — 5层架构图（用 layer-stack 组件）
§4 入口与启动流程   — CLI 入口 + 启动时序图
§5 [核心模块 A]     — 如：数据库连接层
§6 [核心模块 B]     — 如：补全引擎
§7 [核心模块 C]     — 如：配置管理
§8 [核心模块 D]     — 如：特殊命令框架
§9 输出/展示层      — 渲染/格式化
§10 核心设计模式    — 架构模式总结
§11 依赖总览        — 核心/可选依赖表格
§12 全局数据流      — 端到端数据流图
```

每个核心模块节（§5-§9）包含：
- 该模块的 class/struct 核心属性（代码块）
- 关键方法/函数的调用流程（flow 图）
- 重要的设计决策说明

## 论文

### 阅读顺序
1. 标题、摘要、关键词
2. 引言（研究背景、问题定义、本文贡献）
3. 相关工作（快速了解领域）
4. 方法（核心创新）
5. 实验（数据集、指标、结果）
6. 结论与局限性

### 需提取的信息
- 研究背景：解决什么问题，现有方法的不足
- 方法创新：核心思路（用 1-2 句话概括）
- 关键公式：最重要的 2-3 个（用代码块或表格说明）
- 实验结果：关键指标对比（表格）
- 局限性与未来工作

### 推荐文章结构（论文解读）

```
§1 论文概览         — TL;DR（3句话总结）+ 元信息（作者、发表、任务）
§2 研究背景         — 问题定义 + 现有方法的不足
§3 方法详解         — 核心方法，分步骤说明
§4 关键公式解读     — 最重要的数学表达（逐行注释）
§5 实验设置         — 数据集 + 基线方法 + 评价指标
§6 实验结果         — 主结果表 + 关键发现
§7 消融实验         — 各模块贡献分析
§8 总结与展望       — 贡献总结 + 局限性 + 未来方向
```

## 产品/文档

### 阅读顺序
1. 产品概述、核心价值主张
2. 功能列表 / Feature matrix
3. 架构说明（如有）
4. 用户场景 / 使用案例

### 需提取的信息
- 是什么：一句话定义
- 解决什么问题：用户痛点
- 核心功能：5-8 个，每个有具体说明
- 技术架构（如有）
- 与竞品的差异化

### 推荐文章结构（产品/系统介绍）

```
§1 产品定位         — 一句话定义 + 核心价值（Hero 区）
§2 核心功能         — 卡片组，每个功能 1 张卡片
§3 技术架构         — 分层图或架构图
§4 核心概念/术语    — 表格形式，逐一解释
§5 工作流           — 典型使用场景的步骤图
§6 技术实现亮点     — 关键技术选型和设计决策
§7 与竞品对比       — 对比表格
§8 总结             — 适用场景 + 获取方式
```

## 通用写作要点

### Hero 区必须包含
```
项目名称（大标题）
一句话描述（副标题）
元信息标签：版本 · 协议 · 语言版本 · 代码量 · 维护者
```

### 章节标题格式（HTML）
```html
<h2><span class="icon">🔍</span>章节名称</h2>
```
emoji 选择：🧭项目简介 · 📁目录 · 🏗️架构 · 🚀入口 · 🗄️数据库 · 💡补全 · ⚙️配置 · ⚡命令 · 🖨️输出 · 💻REPL · 📦批量 · 🔄后台 · 🔐安全 · 🎨样式 · 🧩设计模式 · 📦依赖 · 🌊数据流

### 代码块标注规则
- Python 文件 → 顶部注释写文件路径：`# sqlexecute.py`
- 流程图用纯文本（空格对齐）或在 `<div class="flow">` 中
- 表格优先用 HTML `<table>` 组件（更易控制样式）
- ASCII 树状结构用 `<div class="tree">` 组件

### 长度参考
| 文章类型 | 目标字数（中文）| 节数 |
|---|---|---|
| 代码库解读 | 8,000-15,000 | 10-14 节 |
| 论文解读 | 4,000-8,000 | 6-8 节 |
| 产品介绍 | 3,000-6,000 | 6-8 节 |
