01

项目概述

Litefuse 是一个开源 LLM 工程平台，帮助团队对 AI 应用进行协作开发、监控、评估和调试。平台覆盖从数据采集到持续迭代的完整 LLM 工程生命周期。

核心能力说明标签
链路追踪（Tracing） 捕获 LLM 调用完整链路，包括输入输出、Token 消耗、耗时、成本 实时核心
自动评估（Evals） LLM-as-judge 自动评估 + 人工标注双模式 AI人工
提示词管理（Prompts） 版本化提示词，支持 production/staging 标签，支持组合 版本化
数据集与实验 从 Trace 构建数据集，运行对比实验，追踪指标变化 实验
协作（Collaboration） 团队内联注释、打分、标注审核队列 团队
集成（Integrations） PostHog、Mixpanel、S3/Azure、Slack、GitHub Actions 生态

WorkerManager 核心机制

  additionalOptions,

核心功能模块

数据追踪层次模型

Trace（一次完整 AI 应用调用）→ 包含多个 Observation（子节点）→ 分三种类型：SPAN（逻辑步骤）、GENERATION（LLM 调用，记录成本/Token）、EVENT（点事件）
Trace 数据结构 
Trace
├── name, projectId, timestamp, tags, environment
├── userId (用户维度), sessionId (会话分组)
├── externalId (外部系统关联)
└── Observation[]
    ├── SPAN     → 逻辑步骤（retrieval / formatting）
    ├── GENERATION → LLM 调用
    │   ├── model, modelParameters
    │   ├── input / output
    │   ├── promptTokens + completionTokens
    │   └── calculatedTotalCost (自动按定价计算)
    └── EVENT    → 点事件（cache-hit / tool-call）

评估工作流

1
创建 EvalTemplate
定义 LLM 评估提示词模板 + 变量映射（input、output、metadata 字段）
2
配置 JobConfiguration
设定过滤条件（哪些 Trace 触发评估）+ 采样率 + 触发延迟时间
3
TraceUpsert → 创建 JobExecution
每当新 Trace 满足过滤条件，系统自动创建一条 JobExecution 记录
4
Worker EvalQueue 消费
拉取 Trace 数据 → 填充模板变量 → 调用配置的 LLM（通过 LlmApiKey）→ 解析结构化输出
5
写入 Score
评估结果（数值/分类/布尔）绑定到原 Trace，source = EVAL

提示词版本化

🔒 Immutable 版本
每次修改产生新版本，历史版本永不修改
🏷️ 标签系统
production / staging 为受保护标签，自定义标签自由添加
🔗 组合依赖
通过 PromptDependency 实现提示词嵌套组合

仪表盘图表类型

LINE 折线图 BAR 柱状图 PIE 饼图 HISTOGRAM 直方图 HEATMAP 热力图 SCATTER 散点图 NUMBER 数值卡片 TABLE 数据表格 PIVOT_TABLE 透视表
